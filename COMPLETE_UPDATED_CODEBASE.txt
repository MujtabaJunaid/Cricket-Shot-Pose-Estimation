DATASET FEATURES SUPPORTED
==========================

Input Formats Handled:
  - Direct landmark arrays
  - Landmark dictionaries
  - Image frames (logged, requires processing)

Output Format:
  - 99 features per sample (33 keypoints × 3 coordinates)
  - Sequence length: 10 frames (padded/truncated)
  - Label: Integer class ID (0-8)

Preprocessing:
  - Automatic normalization
  - StandardScaler fitting
  - Sequence padding/truncation
  - Data validation

TRAINING WORKFLOW WITH HUGGINGFACE
===================================

Step-by-Step:

1. Install Dependencies:
   pip install -r requirements.txt

2. Verify Dataset Access:
   python inspect_hf_dataset.py

3. Train Model:
   python training/run_training.py \
     --mode train \
     --use-huggingface \
     --epochs 50 \
     --batch-size 32 \
     --learning-rate 0.001 \
     --device cuda

4. Export Model:
   python training/run_training.py \
     --mode export \
     --export-format ensemble

5. Start API Server:
   python backend/app.py

6. Use Frontend:
   Open http://localhost:3000

PYTHON API USAGE
================

Loading HuggingFace Dataset Directly:

  from training.dataset import HuggingFaceDataset
  from backend.config import REVERSE_SHOT_CLASSES

  dataset = HuggingFaceDataset(
    dataset_name="rokmr/cricket-shot",
    split="train",
    label_mapping=REVERSE_SHOT_CLASSES,
    sequence_length=10,
    use_temporal=True
  )

  sample_data, label = dataset[0]
  print(f"Sample shape: {sample_data.shape}")
  print(f"Label: {label}")

Using Data Loaders:

  from training.dataset import get_data_loaders
  from backend.config import REVERSE_SHOT_CLASSES

  train_loader, val_loader, dataset = get_data_loaders(
    label_mapping=REVERSE_SHOT_CLASSES,
    batch_size=32,
    validation_split=0.2,
    use_huggingface=True,
    dataset_name="rokmr/cricket-shot"
  )

  for batch, labels in train_loader:
    print(f"Batch shape: {batch.shape}")
    print(f"Labels shape: {labels.shape}")
    break

CONFIGURATION OPTIONS
====================

Environment Variables:

  USE_HUGGINGFACE=true
  HUGGINGFACE_DATASET=rokmr/cricket-shot
  DEVICE=cuda
  BATCH_SIZE=32
  EPOCHS=50

Example .env file:

  USE_HUGGINGFACE=true
  DEVICE=cuda
  BATCH_SIZE=32
  EPOCHS=50

Command-Line Arguments:

  --use-huggingface         # Enable HuggingFace dataset
  --dataset-name <name>     # Dataset identifier (default: rokmr/cricket-shot)
  --data-dir <path>         # Local data directory (for local training)
  --checkpoint-dir <path>   # Model checkpoint directory
  --epochs <int>            # Number of epochs
  --batch-size <int>        # Batch size
  --learning-rate <float>   # Learning rate
  --device {cuda,cpu}       # Training device
  --mode {train,export}     # Training or export mode
  --export-format <fmt>     # Export format

ERROR HANDLING
==============

Dataset Not Found:
  - Automatic download on first use
  - Cached at ~/.cache/huggingface/
  - Check internet connection

Authentication Required:
  huggingface-cli login
  Enter token from https://huggingface.co/settings/tokens

Memory Issues:
  - Reduce batch size
  - Use CPU: --device cpu
  - Reduce sequence_length

Label Mismatch:
  - Check REVERSE_SHOT_CLASSES mapping
  - Verify dataset format
  - See inspect_hf_dataset.py for debug

COMPARISON: HuggingFace vs Local Data
======================================

HuggingFace Advantages:
  ✓ No data collection needed
  ✓ Pre-processed and validated
  ✓ Easy sharing and reproduction
  ✓ Community-maintained
  ✓ Automatic caching
  ✓ Scalable

Local Data Advantages:
  ✓ Custom data
  ✓ Full control over processing
  ✓ Privacy (no upload)
  ✓ Domain-specific optimization

Both Supported:
  - Use --use-huggingface for HuggingFace
  - Omit flag for local data
  - Easy switching between datasets

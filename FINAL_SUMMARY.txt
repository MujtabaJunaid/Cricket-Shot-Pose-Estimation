PYTHON API USAGE

Using HuggingFaceDataset Directly:

  from training.dataset import HuggingFaceDataset
  from backend.config import REVERSE_SHOT_CLASSES
  
  dataset = HuggingFaceDataset(
    dataset_name="rokmr/cricket-shot",
    split="train",
    label_mapping=REVERSE_SHOT_CLASSES
  )
  
  sample, label = dataset[0]
  print(f"Shape: {sample.shape}, Label: {label}")

Using Data Loaders:

  from training.dataset import get_data_loaders
  
  train_loader, val_loader, dataset = get_data_loaders(
    use_huggingface=True,
    dataset_name="rokmr/cricket-shot",
    batch_size=32
  )
  
  for batch, labels in train_loader:
    print(f"Batch: {batch.shape}, Labels: {labels.shape}")

================================================================================
DATASET DETAILS
================================================================================

Dataset Name: rokmr/cricket-shot
Source: https://huggingface.co/datasets/rokmr/cricket-shot

Shot Classes (9 total):
  0: drive
  1: pull
  2: cut
  3: defense
  4: backfoot_play
  5: cover_drive
  6: off_drive
  7: on_drive
  8: lofted_shot

Data Format:
  - Landmarks: 33 keypoints × 3 coordinates = 99 features
  - Sequence: Variable length, normalized to 10 frames
  - Labels: Integer class IDs (0-8)

Processing:
  - Automatic normalization using StandardScaler
  - Sequence padding/truncation
  - Validation and error handling

ARCHITECTURE CHANGES
================================================================================

Before (Local Only):
  User Data → CricketPoseDataset → Training Pipeline

After (Dual Support):
  User Data → CricketPoseDataset ─┐
                                   ├→ get_data_loaders() → Training Pipeline
  HuggingFace → HuggingFaceDataset ┘

Key Design Decisions:
  1. Two separate dataset classes with identical interfaces
  2. Factory pattern in get_data_loaders() for source selection
  3. Centralized label mapping using REVERSE_SHOT_CLASSES
  4. Fallback error handling for missing dependencies

================================================================================
TROUBLESHOOTING
================================================================================

Issue: "ModuleNotFoundError: No module named 'datasets'"
  Solution: pip install datasets huggingface-hub

Issue: "Network error downloading dataset"
  Solution: Check internet connection. Dataset is cached after first download at ~/.cache/huggingface/

Issue: "Authentication error"
  Solution: Run: huggingface-cli login
  Then enter token from: https://huggingface.co/settings/tokens

Issue: "Out of memory"
  Solution: Reduce batch size: --batch-size 16
  Or use CPU: --device cpu

Issue: "Label mismatch"
  Solution: Check REVERSE_SHOT_CLASSES in backend/config.py
  Run: python inspect_hf_dataset.py to verify format

================================================================================
PERFORMANCE CONSIDERATIONS
================================================================================

HuggingFace vs Local:

HuggingFace Advantages:
  + No local storage needed (except cache)
  + Pre-processed and validated data
  + Easy community sharing
  + Automatic versioning
  + Streaming support for large datasets

Local Data Advantages:
  + Faster access (no network)
  + Custom preprocessing
  + Privacy (no upload needed)
  + Full control over data

Speed Comparison:
  - First load (HF): ~5-10 minutes (includes download)
  - Subsequent loads (HF): ~1-2 minutes (cached)
  - Local: ~30 seconds (instant, already preprocessed)

Memory Usage:
  - HF streaming: Lower peak memory, progressive loading
  - Local: Fixed memory based on batch size
